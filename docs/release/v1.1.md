# MDViewer v1.1

v1.1 adds local OCR to the existing Save/Print to Markdown product on macOS 13+ Apple Silicon.

## Added

- Apple Vision OCR for PNG and JPEG images without semantic text metadata.
- Page-selective OCR for scanned and mixed PDFs; digital pages stay on PDFium text extraction.
- Accurate recognition, language correction and automatic language detection on-device.
- Stable `ocr_no_text_found` and `ocr_low_confidence` warnings.
- 200 DPI PDF rendering bounded to 4096 px per dimension, 16 million pixels per page and
  64 million pixels per conversion.
- Cancellation checks before and immediately after each PDF page OCR operation.
- A provider-neutral `mdconvert-ocr` contract that compiles on Windows and can host a future
  Windows/Linux backend without changing the converters.

## Privacy and distribution

OCR uses the system Vision framework and never uploads document bytes. The release includes no
PyTorch, Docling, model download or network conversion. v1.1.0 is published first for Apple Silicon;
Windows and Linux remain source-compatible but do not yet ship a production OCR backend or binary.

## Known limits

- Images with existing semantic text metadata are not OCRed again.
- Images embedded in a PDF page that already has digital text are not OCRed in v1.1.
- OCR preserves detected lines and geometry but cannot recreate every visual layout or semantic
  structure lost by printing.
- Low-confidence text is retained for review instead of being silently discarded.
