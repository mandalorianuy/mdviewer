# MDViewer v1.2

MDViewer v1.2 closes the known OCR fidelity residuals while preserving the local-only conversion
architecture. The first binary remains macOS 13+ Apple Silicon and is released as a Developer ID
signed, notarized and stapled DMG.

## Included

- OCR of eligible textual images embedded in PDF pages that also contain digital text.
- Geometry projection into the embedded image, original asset retention and spatial duplicate
  suppression.
- Explicit icon/decorative-image thresholds and shared bounded pixel budgets.
- Apple Vision on macOS, Windows Media OCR on Windows, and Tesseract 5 through its library API on
  Linux.
- Stable real-backend token-recall fixture plus deterministic contract, geometry, confidence,
  eligibility and duplicate tests.
- No PyTorch, Docling, network conversion, model download or OCR subprocess.

## Platform notes

The public v1.2 binary is Apple Silicon macOS. Windows and Linux core, CLI and desktop remain CI
targets, but this release does not claim published installers or universal print adapters for those
platforms.

Windows uses OCR languages installed in the user profile. The Windows API does not expose
recognition confidence; MDViewer represents it as neutral `0.5` instead of inventing a high or low
value.

Linux requires Tesseract 5, Leptonica, and the `eng` plus `spa` trained-data packages. They are
resolved locally at startup and no document or model bytes are fetched. Binary packaging will
measure and record the actual runtime/model footprint before choosing AppImage, Flatpak, or system
dependencies.

## Remaining distribution work

- Windows virtual printing requires a native Print Support App v4/MSIX and Windows end-to-end
  installation evidence.
- Linux universal printing requires a PAPPL/CUPS Printer Application and a native
  install/print/save/uninstall smoke test.
- These adapters remain separate from the OCR core and are not release claims of v1.2.0.
