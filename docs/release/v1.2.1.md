# MDViewer v1.2.1

MDViewer v1.2.1 packages the v1.2 OCR and conversion product for all three supported desktop build
targets. Conversion remains local-only. On macOS, this release adds a user-local virtual printer
that appears in Chrome and other applications as `MDViewer — Guardar como Markdown`, alongside the
existing native Print dialog PDF workflow.

## Artifacts

- macOS 13+ Apple Silicon: `MDViewer-1.2.1-arm64.dmg`, Developer ID signed, notarized, stapled and
  accepted by Gatekeeper.
- Windows x64: `MDViewer-1.2.1-x64-setup.exe`, per-user NSIS installer signed with Authenticode and
  a trusted timestamp.
- Linux x64: `MDViewer-1.2.1-x86_64.AppImage` and `MDViewer-1.2.1-amd64.deb`, built on Debian 12.
- Commit-bound macOS `package-receipt.json`, `package-receipt-windows-x64.json`, and
  `package-receipt-linux-x64.json` assets plus GitHub build provenance attestations.

## Linux OCR runtime

The AppImage includes Tesseract 5, Leptonica and pinned `tessdata_fast` 4.1.0 English and Spanish
language files. The Debian package declares the equivalent runtime dependencies. Neither package
downloads OCR models while converting documents.

## Verify provenance

After downloading an artifact from GitHub Releases:

```bash
gh attestation verify PATH/TO/ARTIFACT -R mandalorianuy/mdviewer
```

Compare its SHA-256 with the matching platform receipt as an independent integrity check.

## Distribution boundary

These artifacts install the MDViewer desktop application and Save As conversion. The macOS artifact
also includes the installer and lifecycle management for the local CUPS/IPP virtual printer. Windows
Print Support App v4/MSIX and Linux PAPPL/CUPS virtual-print adapters are not claims of v1.2.1.

The Windows artifact remains unpublished until MDViewer is enrolled in SignPath Foundation's
open-source program and the signed candidate passes the full install lifecycle.
