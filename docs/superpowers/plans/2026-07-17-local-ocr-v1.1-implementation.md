# OCR local v1.1 — plan de implementación

Autoridad de diseño:
`docs/superpowers/specs/2026-07-17-local-ocr-v1.1-design.md`.

Cada task se implementa con test rojo observable, cambio mínimo y test verde.

## Task 1 — Contrato OCR portable

- Crear `crates/mdconvert-ocr` y agregarla al workspace.
- Probar tipos, validación de rectángulos/dimensiones, orden y errores estables.
- Implementar `OcrEngine`, `OcrInput`, `OcrLine`, `OcrOutput`, `OcrError` y
  `LocalOcrEngine` no disponible fuera de macOS.
- Verificar compilación host y `cargo check` con un target no macOS instalado si
  el entorno lo permite.

## Task 2 — Backend Vision macOS

- Agregar dependencias `objc2-*` sólo bajo `cfg(target_os = "macos")`.
- Probar Vision contra una imagen fixture con texto grande y estable.
- Implementar `VNRecognizeTextRequest` preciso, corrección lingüística,
  detección automática y resultados ordenados de arriba hacia abajo.
- Redactar errores de framework.

## Task 3 — Imágenes PNG/JPEG

- Probar con motor falso: OCR se invoca sólo cuando corresponde, párrafos antes
  del asset, no-text y baja confianza.
- Inyectar `&dyn OcrEngine` sin romper el conversor portable existente.
- Conectar `LocalOcrEngine` en CLI y actualizar el contrato de warnings/UI.

## Task 4 — Páginas PDF escaneadas

- Probar proyección de coordenadas y síntesis de palabras con motor falso.
- Renderizar sólo páginas sin glifos a resolución y presupuesto acotados.
- Soportar PDFs escaneados y mixtos conservando el extractor/reconstructor
  digital existente.
- Agregar cancelación entre páginas.

## Task 5 — Evidencia de producto

- Agregar fixtures/goldens OCR deterministas y tests de integración CLI/desktop.
- Actualizar parity manifest, README, CLI reference, arquitectura, seguridad,
  release notes y limitaciones.
- Cambiar versiones coordinadamente a `1.1.0`.
- Ejecutar `./scripts/verify-workspace.sh` y pruebas manuales de imagen, PDF
  escaneado, PDF mixto y flujo de Guardar como Markdown.

## Task 6 — Publicación

- Commit, push y PR a GitHub con evidencia del head exacto.
- Integrar por PR y sincronizar el mirror OneDev.
- Construir arm64, firmar, verificar permisos, notarizar con el perfil Keychain
  `mdviewer-release`, staple y validar Gatekeeper.
- Publicar GitHub Release `v1.1.0`, tag firmado si el flujo vigente lo exige,
  checksums y artefacto notarizado.
