# OCR local v1.1 — diseño

Fecha: 2026-07-17

## 1. Resultado de producto

MDViewer v1.1 convierte localmente a Markdown imágenes PNG/JPEG con texto y
PDFs escaneados. En macOS 13+ Apple Silicon usa Vision; no sube documentos, no
invoca servicios externos y no incorpora runtimes de machine learning como
PyTorch o Docling.

La primera publicación de v1.1 continúa siendo macOS Apple Silicon. El core,
el contrato OCR y los conversores permanecen compilables en macOS, Windows y
Linux para poder agregar backends nativos o Tesseract sin cambiar los formatos.

## 2. Alcance

Incluido:

- OCR de PNG y JPEG sin texto semántico suficiente.
- OCR de cada página PDF que no tenga glifos extraíbles.
- PDFs mixtos: conservar texto digital y ejecutar OCR sólo en páginas sin texto.
- Orden de lectura y geometría normalizada de las líneas reconocidas.
- Reconocimiento preciso, corrección lingüística y detección automática de
  idioma cuando el backend lo soporte.
- Límites explícitos de páginas y píxeles renderizados.
- Diagnósticos estables para backend no disponible, OCR sin texto y resultados
  de baja confianza.
- Composición automática desde CLI y desktop en macOS.

Fuera de v1.1:

- OCR de imágenes embebidas en una página PDF que ya contiene texto digital.
- Edición interactiva de regiones, idioma o confianza.
- Procesamiento en red, OCR generativo o reconstrucción visual pixel-perfect.
- Backend OCR productivo para Windows o Linux.

## 3. Arquitectura

Se agrega `mdconvert-ocr`, una crate portable con:

- `OcrEngine`, contrato síncrono, `Send + Sync`, sin dependencias de UI.
- `OcrInput`, bytes codificados, media type y dimensiones comprobadas.
- `OcrOutput` y `OcrLine`, texto, confianza y rectángulo normalizado.
- `LocalOcrEngine`, backend de composición: Vision en macOS y estado no
  disponible tipado en otras plataformas.

Las dependencias Objective-C/Vision se declaran sólo para `target_os =
"macos"`. `mdconvert-core` no conoce APIs de plataforma. `mdconvert-formats` y
`mdconvert-pdf` reciben un `&dyn OcrEngine`; `mdconvert-cli` elige el backend en
la raíz de composición. El desktop continúa usando el contrato CLI existente.

## 4. Imágenes

El parser PNG/JPEG actual sigue autenticando estructura, dimensiones, límites y
metadata antes de OCR. Si no hay texto semántico, el conversor entrega al motor
los mismos bytes ya validados. Las líneas no vacías se convierten en párrafos en
orden visual y la imagen original se conserva como asset después del texto.

Si no se reconoce texto, la conversión conserva la imagen y emite
`ocr_no_text_found`. Si el backend no está disponible, conserva el comportamiento
portable con `ocr_deferred`. Una línea de confianza inferior a 0.5 se conserva,
pero produce `ocr_low_confidence`; no se borra silenciosamente contenido.

## 5. PDFs

PDFium extrae primero cada página como hoy. Una página con texto digital no se
renderiza para OCR. Una página sin texto se renderiza a 200 DPI, con un máximo
de 4096 px por dimensión y 16 millones de píxeles. El presupuesto acumulado de
la conversión es 64 millones de píxeles y nunca puede exceder `max_pages`.

Las líneas Vision, expresadas en coordenadas normalizadas, se proyectan a la
geometría efectiva de la página y se incorporan como palabras posicionadas. La
reconstrucción existente decide párrafos y orden, evitando un segundo pipeline
de Markdown. Si ninguna página contiene texto digital ni texto reconocido, el
resultado continúa siendo `ocr_required`.

## 6. Privacidad, errores y cancelación

Vision se ejecuta on-device. No se agrega permiso de usuario ni entitlement. Los
mensajes públicos no incluyen contenido del documento. Errores internos del
framework se reducen a diagnósticos tipados y redactados.

La cancelación se comprueba antes y después de la conversión. Un
hook por página queda en el contrato de extracción para evitar iniciar el OCR de
la página siguiente después de cancelar; una llamada Vision ya iniciada no se
interrumpe de forma insegura.

## 7. Criterios de aceptación

- Un fixture PNG/JPEG real reconoce texto y produce Markdown más el asset.
- Un PDF completamente escaneado produce Markdown, no `ocr_required`.
- Un PDF mixto conserva el texto digital y agrega exactamente una vez el texto
  de las páginas escaneadas.
- Una imagen sin texto se conserva con `ocr_no_text_found`.
- Los tests con motor falso prueban contrato, geometría, confianza y límites sin
  depender de Vision.
- Un test macOS ejecuta Vision con un fixture estable.
- El verificador completo, el build desktop y el parity manifest pasan.
- El binario firmado/notarizado se prueba desde un `.app` limpio antes de
  publicar `v1.1.0`.
